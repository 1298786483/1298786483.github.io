项目简介

一键场地预约 v4.0（文件名示例 0.9.py）是一个基于 Selenium 的自动化脚本，目标是帮助用户在东北大学（示例站点 book.neu.edu.cn）抢/预约体育场地。项目提供了带交互式 Tkinter GUI 的桌面启动器，并集成了邮件与 PushPlus 推送通知、日志输出和简单的防多开机制（锁文件 + 进程检查）。脚本尽量保留人工可配置项（时间段、场地号、刷新间隔、抖动、验证码支持等），并对常见页面差异与弹窗做了稳健处理。

功能亮点

  • Selenium 自动登录并在页面上寻找可预约时段
  • 支持多个目标时段和多个场地同时监控
  • 支持验证码（可在启动时填写验证码）
  • 预约提交后，自动检测是否进入支付页或直接进入预约成功页
  • 支持邮件通知（SMTP）和 PushPlus 推送（可发送 HTML 日志）
  • GUI（Tkinter）支持：填写账号、目标时段、场地、邮件与 PushPlus 配置、自动倒计时及手动启动
  • 日志文件保存在 logs/，每次运行生成独立日志文件
  • 多开检测（logs/booking_app.lock + 进程扫描）
  • ANSI 色彩转 HTML 用于把彩色日志发送到推送/邮件（便于阅读）

运行环境与依赖

推荐在 Python 3.8+ 环境下运行。主要第三方依赖：

  • selenium
  • psutil
  • colorama
  • requests（可选，用于 PushPlus）
  • tkinter（Python 自带 GUI 库）
  • 如果发送邮件：smtplib（Python 标配）

另外需要匹配你 Chrome 浏览器版本的 ChromeDriver，可从 https://chromedriver.chromium.org/ 下载并把驱动放在 PATH 或与脚本同目录。

准备工作

  1. Chrome / ChromeDriver

    • 查 Chrome 版本：chrome://settings/help
    • 下载对应版本的 chromedriver 并解压放到系统 PATH 或脚本目录。
    • Windows 用户可能需要把 chromedriver.exe 放到脚本同目录或 C:\Windows\。

  2. 账号信息（NEU 预约站点）

    • 确保账号能正常在浏览器中登录。
    • 若站点有验证码，脚本提供了“手动在启动时填写验证码并提交”的方式（会等待并填写 ID 为 PM1 的验证码输入框，脚本会尽量兼容页面差异）。

  3. SMTP（若使用邮件通知）

    • 准备 SMTP 服务器/账号/密码与接收邮箱。
    • 脚本使用 TLS（server.starttls()），端口通常 587。

  4. PushPlus（可选）

    • 如果要使用 PushPlus，登录 PushPlus 获取 token，并在配置中填入 PUSHPLUS_TOKEN。

配置说明（建议使用 config.json）

脚本会读取同目录下的 config.json（如果不存在，脚本会使用默认并可以通过 GUI 保存）。下面给出一个完整示例，供保存为 config.json 使用：

字段说明（重点）：

  • USERNAME / PASSWORD：登录站点的账号/密码
  • VERIFICATION_CODE：如果你想预先填写验证码可放这里（通常留空，GUI 会在必要时填写）
  • MEMBER_NAME / MEMBER_ID：预约人姓名与证件（会填入表单）
  • TARGET_SLOTS：目标时间段的字符串列表，匹配页面显示的时段文本（例如 “18:00-20:00”）
  • SELECTED_COURTS：场地编号列表（字符串）
  • REFRESH_TIME：默认刷新间隔（秒）
  • JITTER_TIME：刷新抖动（秒）
  • SMTP_*：邮件通知配置（选填）
  • PUSHPLUS_TOKEN：PushPlus 通知 token（选填）
  • SHOW_BROWSER_DEBUG：是否显示浏览器窗口（True 显示，False headless）

将上述保存为 config.json，或通过 GUI 在运行后保存配置。

启动与使用

GUI 启动（推荐）

双击或在命令行运行：

程序会弹出 Tkinter 界面，你可以：

  1. 填写账号/密码、会员姓名/证件
  2. 勾选目标时段与场地
  3. 填写 SMTP 或 PushPlus（可选）
  4. 设置刷新间隔与抖动
  5. 选择“自动倒计时启动”（倒计时结束后自动启动预约）
  6. 点击“开始预约”手动启动

GUI 会在开始前把配置保存到 config.json（调用 save_config）。

命令行 / 自动启动

如果想用 CLI 启动（例如通过 systemd 或计划任务），可以构建一个简短 wrapper 脚本，读取 config.json 并直接调用 start_booking_process(cfg)。但注意 GUI 相关代码会在 __main__ 中创建应用；若不想打开 GUI，可修改或写一个新的启动脚本只调用 start_booking_process。

关键模块与代码说明（逐个讲解）

下面按脚本中的函数与机制，解释其作用与实现要点。

ANSI 与日志可视化：ansi_to_html_br / ansi_to_html

  • 目的：把带 ANSI 颜色的日志转换为 HTML（带颜色）或转换为 HTML 中可显示的片段，便于把日志以 HTML 邮件或通过 PushPlus 发送时保持颜色与换行。
  • ansi_to_html_br(text)：把 ANSI 颜色序列解析为对应的内联 style="color:...;" 的 span，并把换行替换为 <br/>，最后包在 <div>。
  • ansi_to_html(s)：另一种更简单的实现，先把 ESC 转为统一标记后逐个替换颜色，适合内嵌在 <pre>。

这两者用于构建邮件/PushPlus 的 HTML 内容，让接收方能看到彩色日志（便于排查）。

通知：send_pushplus(token, title, content, channel="mail", webhook="")

  • 功能：调用 PushPlus API（脚本使用 http://www.pushplus.plus/send/{token}）发送通知。
  • 特点：优先使用 requests，若不可用退回 urllib；返回 (ok: bool, resp_text: str) 方便上层逻辑根据返回值判断是否需要重试或报警。
  • 使用场景：预约成功/失败通知、异常提醒。

示例集成处：预约成功后会把本轮日志转为 HTML 并通过 PushPlus 发送。

注意：PushPlus 的 API 与使用策略可能随时间变化，请以官方文档为准并替换 URL 或参数（本脚本按常见实现保留了兼容处理）。

邮件通知：send_email(...)

  • 功能：通过 SMTP（启用 STARTTLS）发送邮件，支持 plain 或 html 内容。
  • 使用注意：
    • SMTP 服务、端口与账号需正确（例如：Gmail 需 app password 或 OAuth2，不建议在公开仓库保存真实密码）
    • 脚本使用 MIMEText 设置 utf-8 编码，主题支持中文

预约成功或异常会把日志打包为 HTML 邮件发送到 RECIPIENT_EMAIL。

浏览器初始化：init_driver(username, password, verification_code=None, show_browser_debug=True)

  • 功能：创建 Chrome WebDriver，打开登录页面 http://book.neu.edu.cn/booking/page/rule/stadium/ypg_ymb，填写用户名（ID un）和密码（ID pd），并尝试点击登录按钮（ID index_login_btn）。
  • 参数：
    • show_browser_debug 控制是否以 headless 模式运行（False 会尝试 --headless=new 或旧的 headless 设置）
  • 兼容性与容错：
    • 包含显式等待（WebDriverWait）以确保元素可见
    • 如果提供 verification_code，会尝试等待 ID PM1 的验证码框并填写
  • 返回：登录后的 driver（未继续自动点击场地，便于后续逻辑独立处理）

注意：页面元素 ID/类名随网站变动，这里需要根据页面实际 DOM 适时调整选择器。

可用性读取：get_availability_str(driver)

  • 功能：抓取页面上若干带时间段的 panel，读取每个 TimeDiv 下的 li 文本，汇总成日志字符串。
  • 特殊处理：当至少存在一个可用时段（含关键字 “可用”、”可预约”）时，会对时间段部分染色（使用 ANSI \033[92m 亮绿）以便日志突出显示。
  • 输出：一个多行字符串，适合写入日志或推送。

预约尝试：check_and_book_court(driver, court_no, target_slots, member_name, member_id)

  • 功能：在给定 court_no（1-base）对应 panel 内循环寻找 li，当某个 li 同时包含目标时段字符串且包含 “可用” 时触发预约流程：
    • 调用页面 JS（openBook(...)）以打开预约面板
    • 通过 DOM 操作选中具体时段元素、填写姓名和证件号
    • 点击 enter_button 提交
    • 尝试点击 “我同意”（input#read_already）
    • 点击 “去支付”（input#goPay）并在点击后 6 秒内轮询判断是否进入支付页（通过 URL / DOM 检测），或检测无需支付的预约成功页面（如 id=book_no 等）
  • 返回：发生付款页或成功页时返回 (court_no, chosen)，否则返回 None
  • 关键点：保险的页面跳转/弹窗/新窗口检测逻辑（检查 window_handles、current_url、特定 DOM 元素）

页面结构差异或网站更新会影响选择器（如 openBook()、goPay 等），需要你根据实际页面适配。

主循环：start_booking_process(config, max_refresh=10000)

  • 功能：核心逻辑循环：
    • 基于 config 初始化 driver
    • 在合适时间（is_within_booking_time()）执行循环
    • 在特定 RESET_WINDOWS 时间段强制重登（避免被网站踢出/Session 过期）
    • 每次轮询记录日志 log_buffer 并调用 get_availability_str + check_and_book_court
    • 发现成功时：保存日志片段、发邮件、发 PushPlus、处理弹窗
    • 出错时：异常捕获、刷新页面、必要时重登
    • 刷新间隔根据 REFRESH_TIME 或 FAST_REFRESH_WINDOWS 自动调整并加入抖动（JITTER_TIME）
  • 退出/清理：finally 内关闭 driver 并 sys.exit(0)

日志与锁机制

  • 日志文件保存到 logs/，文件名以时间戳命名（YYYYmmdd_HHMMSS.log）
  • 多开检测：
    • 使用文件 logs/booking_app.lock 写入当前 PID
    • 启动时如果检测到该 PID 对应进程仍运行（并且看起来是 python 相关），程序会弹窗提示并退出
    • 额外遍历系统进程寻找可疑命令行（脚本名关键字）并阻止多开
  • 如果程序异常退出，atexit 注册会尝试清除锁文件

常见问题与排错

  1. 登录失败（页面结构变动 / 选择器失效）

    • 检查 init_driver 中查找用户名/密码输入框的 ID（目前脚本使用 un / pd）。打开页面检查真实 DOM 并修改。
    • 通过浏览器手动登录一次看是否有额外安全校验（Captcha / 二次验证）。

  2. ChromeDriver 错误

    • 报错示例：selenium.common.exceptions.WebDriverException: Message: 'chromedriver' executable needs to be in PATH
    • 解决：下载匹配 Chrome 版本的 chromedriver，并放到 PATH 或脚本目录；Windows 下记得添加 .exe 后缀。

  3. 页面元素找不到 & StaleElementReferenceException

    • 网站使用动态加载，增加等待时间或改用 WebDriverWait(driver, timeout).until(EC.element_to_be_clickable(...))
    • 捕获 StaleElementReferenceException 并重试

  4. 验证码问题

    • 脚本仅支持主动填写验证码（你需在 GUI 提供验证码并点提交）
    • 如需自动识别验证码需集成 OCR 服务（例如 tesseract 或第三方验证码识别服务），这超出本脚本范围

  5. 邮件发送失败

    • 检查 SMTP 参数、端口、是否需要 app password（Gmail/QQ 邮箱），脚本默认使用 STARTTLS
    • 查看日志 logs/*.log 中的异常堆栈用于定位

  6. PushPlus 失败

    • 脚本会记录 PushPlus 返回值，查看 log 中 PushPlus 相关条目
    • 如果 requests 不可用会回退到 urllib，也可能在网络层被防火墙阻止

  7. 本地调试看不到“线上效果”

    • mapping 类问题或网站对 localhost 限制。建议先在浏览器手动测试（登录/预约流程），确认选择器/按钮 ID 是你脚本使用的一致。

进阶使用

  1. 作为系统服务运行（Linux / systemd）
    • 写一个小 systemd unit，把 virtualenv 激活并运行该脚本；注意 GUI 模式需桌面环境，系统服务通常以 headless 模式运行。
  2. 作为计划任务（Windows Task Scheduler / cron）
    • Windows：用 Task Scheduler 启动 python 0.9.py（若需要 GUI 则配置为交互式）
    • Linux：写一个 wrapper 脚本定时运行（并把 Xvfb 与 headless 配置好）
  3. 日志轮转
    • 脚本按启动生成 log 文件，可结合 logrotate（Linux）或自定义脚本周期清理 logs/。
  4. 增强容错
    • 可结合外部守护进程（supervisor / systemd）确保脚本崩溃后自动重启
  5. 通用化与抽象
    • 将场地选择器、预约流程抽象成配置化的 selectors.json，方便适配不同学校的系统。

隐私 / 合法性 与 使用协议

  • 本脚本仅作学习与研究用途。请遵守目标站点的使用条款与学校的预约规则。切勿用于商业化或以违反网站条款的方式运行。
  • 不要在公共仓库中放置真实账号/密码、邮件密码或 PushPlus token。使用时请把敏感信息放到本地不被跟踪的配置文件中（例如 .gitignore）。
  • 如果你要对外分享脚本，请移除敏感信息并提醒使用者自行配置。

致谢与贡献

  • 脚本使用 Selenium 做 DOM 操作和自动化浏览器交互，感谢 Selenium 社区与官方文档。
  • 若你有改进建议、适配其他学校网站或增加验证码识别能力，欢迎 fork 并提 PR / issue。
  • 若需我把 0.9.py 中的某个具体函数重构为独立模块、或把 GUI 与逻辑分离为 CLI + service，我可以基于你的需求给出改造方案和示例代码。

快速复核清单（运行前务必查看）

  • 已安装 Python 3.8+ 且激活 virtualenv
  • 安装：pip install selenium psutil colorama requests
  • 下载并放置匹配 Chrome 版本的 chromedriver
  • 填写 config.json（或通过 GUI 填写并保存）
  • 如果使用邮件通知，确认 SMTP 可用
  • 如果使用 PushPlus，确认 token 可用
  • 运行前确保没有现存的 logs/booking_app.lock（若存在请确认对应 PID 是否还在运行）
